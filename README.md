# Daily Reporter

업무 시간을 기록하고 일일/주간 보고서를 자동 생성하는 Windows 데스크탑 앱입니다.

---

## 기능

- **출근/퇴근 기록** — 클릭 한 번으로 타임스탬프와 함께 로그 파일에 저장
- **실시간 시계** — 현재 시각을 메인 화면에 표시
- **정시 알림** — 설정한 알림 간격(30분 / 1시간 / 2시간)마다 알림음과 함께 업무 내용 입력 팝업 표시 (`Enter`로 제출); 알림 오프셋(0분 / 5분 / 10분)을 선택하면 알람 시각보다 그만큼 일찍 팝업이 뜨고, CSV에 기록되는 시각은 알람 시각 그대로
- **마지막 기록 표시** — 가장 최근 기록을 메인 화면에 표시, 파일 외부 수정 시 자동 반영
- **복사 버튼** — 마지막 기록의 업무내용을 클립보드에 복사
- **로그 확인** — 오늘 로그 파일을 기본 앱으로 바로 열기
- **일일보고 생성** — 내장 템플릿 기반으로 당일 로그를 채운 hwpx 파일 자동 생성; 첫 항목 시작 시각은 근무 시간대 설정값(미설정 시 09:00) 사용; 점심 시간 항목도 보고서에 포함되며, 근무 시간 계산 시 설정된 점심 시간대만큼 자동 제외
- **주간보고 생성** — 내장 템플릿 기반으로 기간 내 로그를 집계한 hwpx 파일 자동 생성
- **점심 시간 자동 기록** — 설정한 점심 시간대를 종료하는 알람 슬롯은 팝업 없이 자동으로 `점심 시간` 로그 기록 (각 슬롯은 직전 구간을 표현하므로 점심 종료 시각이 기록됨; 예: 점심 12:00~13:00 + 1시간 간격 → 13:00 슬롯 자동 기록); 기록 시각은 알람 시각 그대로
- **근무 시간대 기반 알람 스케줄 정렬** — 근무 시간대를 설정하면 근무 시작 시각을 anchor로 삼아 알림 간격만큼씩 떨어진 슬롯에 알람이 울림 (근무 시작 시각 자체는 알람에서 제외); 예: 9-6 + 간격 1시간 → 10:00, 11:00, … / 9-6 + 간격 2시간 → 11:00, 13:00, … / 9-6 + 간격 30분 → 09:30, 10:00, …
- **출근 상태 자동 복원** — 앱 재시작 시 오늘 로그에 출근 기록이 있으면 알람 자동 재개
- **자동 시작** — 윈도우 부팅 시 자동 실행 설정/해제 (설정 메뉴)
- **저장 경로 설정** — 로그 파일, 일일업무보고, 주간업무보고의 저장 폴더를 각각 독립적으로 지정 가능 (미설정 시 exe 위치 폴더에 저장)

**설정 화면 항목:**

| 항목 | 설명 |
|------|------|
| 윈도우 시작 시 자동 실행 | 부팅 시 앱 자동 실행 (레지스트리 등록) |
| 이전 내용 자동 입력 | 팝업 열릴 때 직전 PROJECT명·업무내용 자동 채움 |
| 시작 시 출근 상태 자동 복원 | 재시작 시 오늘 로그에 출근 기록이 있으면 알람 재개 |
| 알림 간격 | 알람이 울리는 주기 선택 (30분 / 1시간 / 2시간, 기본 1시간); 근무 시간대 설정 시 그 시각을 anchor로 슬롯이 떨어지고(근무 시작 시각 자체는 제외), 미설정 시 자정(00:00) anchor 기준 (30분 → 매시 :00/:30, 1시간 → 매시 정각, 2시간 → 짝수시 정각) |
| 알림 오프셋 | 알람 시각보다 몇 분 일찍 팝업을 띄울지 선택 (0분 / 5분 / 10분, 기본 0분); 기록되는 CSV 시각은 알람 시각 그대로<br>• **0분**: 알람 시각과 동시에 팝업 (예: 알람 10:00 → 팝업 10:00 / 기록 10:00)<br>• **5분**: 알람 시각 5분 전에 팝업 (예: 알람 10:00 → 팝업 09:55 / 기록 10:00)<br>• **10분**: 알람 시각 10분 전에 팝업 (예: 알람 10:00 → 팝업 09:50 / 기록 10:00) |
| 근무 시간대 | 근무 시작/종료 시각 설정 (9-6, 10-7 프리셋 또는 직접 입력); 설정 시 알람 스케줄이 근무 시작 시각을 anchor로 정렬됨 (근무 시작 시각 자체는 알람에서 제외)<br>• 예: 9-6 + 1시간 간격 → 첫 알람 10:00<br>• 예: 9-6 + 2시간 간격 → 첫 알람 11:00<br>• 예: 9-6 + 30분 간격 → 첫 알람 09:30 |
| 점심 시간 | 알림 팝업 대신 자동으로 `점심 시간` 기록할 시간대 설정 (HH:MM 형식, 비워두면 미사용); 점심 종료 시각에 해당하는 알람 슬롯이 자동 기록됨 (각 슬롯이 직전 구간을 표현하기 때문); 기록 시각은 알람 시각 그대로<br>• 예: 점심 12:00 ~ 13:00 + 1시간 간격 → 13:00 슬롯이 `12:00 ~ 13:00 점심 시간`으로 자동 기록<br>• 예: 점심 12:00~13:00 + 30분 간격 → 12:30, 13:00 슬롯이 점심으로 자동 기록 |
| 보고자 성명 | 보고서에 삽입될 이름 (미설정 시 최초 실행 때 자동으로 설정 화면 표시) |
| 로그 파일 저장 경로 | 날짜별 CSV 로그 파일을 저장할 폴더 (기본값: exe 위치) |
| 일일업무보고 저장 경로 | 일일업무보고 hwpx 파일을 저장할 폴더 (기본값: exe 위치) |
| 주간업무보고 저장 경로 | 주간업무보고 hwpx 파일을 저장할 폴더 (기본값: exe 위치) |

## 다운로드 및 실행

1. [Releases 페이지](../../releases/latest)에서 `daily_reporter.exe` 다운로드 후 원하는 폴더에 저장 (단일 파일, 별도 설치 불필요)
2. `daily_reporter.exe` 실행
3. 최초 실행 시 설정 화면이 자동으로 열리면 **보고자 성명** 입력
4. **출근** 버튼 클릭 → 출근 시각 기록 + 정시 알림 시작
5. 설정한 알림 간격마다 팝업이 뜨면(오프셋만큼 알람 시각보다 일찍 표시됨) PROJECT명·업무내용 입력 후 **기록** 또는 `Enter`
6. **퇴근** 버튼 클릭 → 퇴근 시각 기록 + 알림 종료
7. **일일보고 생성** 버튼 → 당일 보고서 hwpx 파일 자동 생성 및 열기

> 로그 파일과 생성된 보고서는 기본적으로 `daily_reporter.exe`가 위치한 폴더에 저장됩니다.  
> 설정 화면에서 로그·일일보고·주간보고 저장 경로를 각각 다른 폴더로 지정할 수 있습니다.  
> 템플릿 파일은 EXE에 내장되어 있어 별도 파일이 필요 없습니다.

**생성 파일명:**

| 구분 | 파일명 형식 | 예시 |
|------|------------|------|
| 일일업무보고 | `유타렉스_일일업무보고_YYMMDD.hwpx` | `유타렉스_일일업무보고_260519.hwpx` |
| 주간업무보고 | `유타렉스_주간업무보고_YYMMDD.hwpx` | `유타렉스_주간업무보고_260519.hwpx` |

## 로그 파일 형식

날짜별로 `YYYY-MM-DD.csv` 파일로 저장됩니다. 4개 열(date, time, project, description).

```
2026-05-12,09:03:52,출근,
2026-05-12,10:00:00,피플카운터,ESP32 BLE 스캔 최적화
2026-05-12,11:00:00,"피플카운터, 집합수요",파라미터 튜닝
2026-05-12,12:00:00,피플카운터,오전 마무리
2026-05-12,13:00:00,,점심 시간
2026-05-12,14:00:00,피플카운터,오후 작업
2026-05-12,18:07:31,퇴근 (자동),
```

프로젝트명에 쉼표가 포함되면 자동으로 큰따옴표로 감싸집니다.

## 빌드

```bash
pip install pyinstaller
pyinstaller daily_reporter.spec
```

빌드 결과물은 `dist/daily_reporter.exe`에 생성됩니다.  
템플릿은 `_templates.py`에 base64로 내장되어 있으며, 빌드 시 별도 파일 없이 EXE 단독으로 배포 가능합니다.

## 기여 방법

버그 리포트, 기능 제안, 코드 기여 모두 환영합니다.

### 버그 리포트 / 기능 제안

[Issues](../../issues) 탭에서 새 이슈를 열어 주세요.  
버그의 경우 **재현 절차**, **기대 동작**, **실제 동작**, **OS 및 Python 버전**을 함께 적어 주시면 빠른 해결에 도움이 됩니다.

### 코드 기여 절차

1. 이 레포를 **Fork**
2. 로컬에 클론 후 개발 환경 설정
   ```bash
   git clone https://github.com/<your-id>/DailyReporter.git
   cd DailyReporter
   pip install pyinstaller   # 빌드에만 필요
   ```
3. 기능/수정 단위로 브랜치 생성
   ```bash
   git checkout -b feat/기능명   # 새 기능
   git checkout -b fix/버그명    # 버그 수정
   ```
4. 변경 사항 구현 및 동작 확인
   ```bash
   python daily_reporter.py   # 소스 직접 실행으로 빠르게 확인
   ```
5. 커밋 — [Conventional Commits](https://www.conventionalcommits.org/) 형식을 따릅니다

   | 접두어 | 용도 |
   |--------|------|
   | `feat:` | 새 기능 추가 |
   | `fix:` | 버그 수정 |
   | `refactor:` | 동작 변경 없는 코드 개선 |
   | `docs:` | 문서(README 등) 변경 |
   | `chore:` | 빌드 스크립트, 의존성 등 기타 변경 |

   ```bash
   git commit -m "feat: 주간보고 기간 선택 UI 추가"
   ```

6. Fork한 레포에 push
   ```bash
   git push origin feat/기능명
   ```
7. 원본 레포로 **Pull Request** 생성 — PR 설명에 변경 이유와 테스트 방법을 간략히 적어 주세요

### 템플릿(hwpx) 수정

`_templates.py`의 base64 문자열을 직접 편집하면 hwpx 구조가 깨질 수 있습니다.  
아래 순서를 권장합니다.

1. 기존 EXE 또는 별도 도구로 hwpx 파일을 추출·수정
2. 수정한 hwpx 파일을 base64로 인코딩하여 `_templates.py`에 반영  
   ```python
   import base64, pathlib
   data = pathlib.Path("template.hwpx").read_bytes()
   print(base64.b64encode(data).decode())
   ```
3. `python daily_reporter.py`로 보고서 생성 결과 확인 후 PR 제출

### 코드 스타일

- 들여쓰기: 공백 4칸
- 문자열: 작은따옴표 (`'`) 기본
- 별도 린터 설정은 없으나, 기존 코드의 스타일을 따라 주세요
